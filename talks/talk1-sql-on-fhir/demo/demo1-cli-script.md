# Demo 1 — CLI Script

**Time budget:** 3 minutes  
**Working directory:** `talks/talk1-sql-on-fhir/demo/data/`  
**Pre-demo:** `cd` to that directory. Confirm `ignixa-sqlonfhir --version` prints a version number.  
**Terminal font:** 24pt minimum. High-contrast theme.

---

## Beat 1: Preview schema — no data (~30s)

**Say:** "Let's start with `preview`. This shows the schema the ViewDefinition will produce — without touching any input data. Great for validating in CI."

```bash
ignixa-sqlonfhir r4 preview --views patient-view.json
```

**Expected output:**
```
=== Schema ===

  birth_date   date
  family_name  string
  gender       string
  given_name   string
  id           string

Resource: Patient  |  FHIR: R4  |  Columns: 5
```

---

## Beat 2: Preview with sample rows (~30s)

**Say:** "Add `--input` to pull sample rows through the evaluator."

```bash
ignixa-sqlonfhir r4 preview --views patient-view.json --input patients.ndjson --rows 5
```

**Expected output (columns are alphabetical; names match the pre-generated patients.ndjson):**

## Beat 3: Validate (~20s)

**Say:** "Before running in a pipeline, validate the ViewDefinition. This catches FHIRPath errors without touching data."

```bash
ignixa-sqlonfhir r4 validate --views patient-view.json
```

**Expected output:**
```
✓ Valid JSON format
✓ Resource type is ViewDefinition
✓ ViewDefinition parsed successfully
  resource=Patient  columns=5  selects=1

✓ ViewDefinition is valid
```

---

## Beat 4: Run — single file to Parquet (~30s)

**Say:** "Now run it. The output format is inferred from the file extension."

```bash
ignixa-sqlonfhir r4 run \
  --views patient-view.json \
  --input patients.ndjson \
  --out patients.parquet
```

**Expected output:**
```
✓ Using FHIR R4 — 5 columns
✓ 20 rows → patients.parquet (2,470 bytes) in 0.4s
```

**Say:** "Open it." [Open in DuckDB or any Parquet viewer]

```bash
# If DuckDB is available:
duckdb -c "SELECT * FROM 'patients.parquet' LIMIT 5;"
```

---

## Beat 5: Batch mode (~45s)

**Say:** "In a real pipeline you have multiple ViewDefinitions and multiple resource types. Batch mode handles all of them in one command."

```bash
ignixa-sqlonfhir r4 run \
  --views views/ \
  --input fhir-ndjson/ \
  --out output/ \
  --format parquet \
  --stats-out output/stats.json
```

**Expected output:**
```
✓ Found 3 ViewDefinition(s)  [R4]

  [1/3] condition-view                                  20 rows  condition-view.parquet (0.0 MB)  0.2s
  [2/3] encounter-view                                 376 rows  encounter-view.parquet (0.0 MB)  0.1s
  [3/3] patient-view                                    20 rows  patient-view.parquet (0.0 MB)  0.0s

✓ Done: 3 completed, 0 skipped, 0 failed
```

**Say:** "Three ViewDefinitions. Three Parquet files. 376 encounters flattened. All in one command — no custom code, no ETL pipeline, no JVM."
