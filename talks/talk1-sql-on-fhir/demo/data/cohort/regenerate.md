# Regenerating the diabetic cohort

The data in `fhir-ndjson/` is **committed**, so the demo works out of the box — you do
*not* need to run this on the day. This documents how the cohort was produced, entirely
with `ignixa-fakes` (FhirFakes v0.5.0), so it's reproducible and auditable.

> Re-running produces a *different* cohort (the `scenario` command isn't seedable), so the
> committed files are the canonical demo data. Regenerate only if you want fresh patients.

## 1. Generate the scenario bundles

12 diagnosed type-2 diabetics (each with a diabetes `Condition` + three HbA1c labs over
six months) and 6 healthy patients (a wellness panel, **no** diabetes, **no** HbA1c — they
exist so the join visibly filters them out):

```powershell
$tmp = "$env:TEMP\cohortgen"
Remove-Item $tmp -Recurse -Force -ErrorAction Ignore; New-Item -ItemType Directory $tmp | Out-Null
1..12 | ForEach-Object { ignixa-fakes r4 scenario DiabeticPatient --out $tmp --resolved-references }
1..6  | ForEach-Object { ignixa-fakes r4 scenario WellnessVisit  --out $tmp --resolved-references }
```

## 2. Split the bundles into per-resource-type NDJSON

`scenario` emits batch **bundles**; SQL on FHIR batch mode wants one NDJSON file per
resource type. (The `population` command has a `--ndjson` flag that does this directly, but
`population` doesn't generate Observations — hence the scenario route + this split step.)

```powershell
python split-bundles.py $env:TEMP\cohortgen ./fhir-ndjson
```

`split-bundles.py` reads every `*.json` bundle in the input folder and writes
`Patient.ndjson`, `Condition.ndjson`, `Observation.ndjson`, `Encounter.ndjson`, and
`MedicationRequest.ndjson` to the output folder.

## What's here

| Path | Contents |
|------|----------|
| `fhir-ndjson/` | The generated FHIR data (18 patients, committed) |
| `views/` | Three ViewDefinitions: `patient-view`, `condition-view`, `observation-view` (HbA1c) |
| `diabetic-cohort.sql` | The DuckDB join query (the demo payoff) |
| `output/` | Parquet produced by the SQL on FHIR CLI — gitignored, regenerated by the demo |
